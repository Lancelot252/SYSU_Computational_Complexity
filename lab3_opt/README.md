# lab3_opt - 优化版本

本目录为 `lab3` 项目的副本，包含小幅性能/可靠性优化和更多测试用例。

主要改动（在文件中以 `# OPTIMIZED:` 标记）：
- `src/max3sat/solver.py`: 增加 `max_attempts` 上限，支持可选的局部爬山优化以更快收敛。
- `src/metric_tsp/solver.py`: 对暴力枚举求解加入回溯剪枝以加速小规模最优解搜索。

如何运行（在 `lab3_opt` 根目录下）：

```bash
python src/main.py --all
```

已新增额外测试用例位于 `test/`。

任务一结论：
- `lab3` 原始版本在随机配置测试中已经很快：例如 `n=50,m=200`、`n=100,m=400` 等常常只需 1-2 次随机循环就达到阈值，并且运行时间在 0.001 秒级别。
- `lab3_opt` 优化版本在相同随机配置下能显著提高满足子句数：
  - `n=50,m=200`：从 178/200 提升到 193/200
  - `n=100,m=400`：从 351/400 提升到 390/400
  - `n=150,m=600`：从 533/600 提升到 584/600
  - `n=250,m=1000`：从 875/1000 提升到 976/1000
- 代价是额外时间开销增加：`lab3` 的时间在 0.0002–0.0012 秒，而 `lab3_opt` 在较大规模时增加到 0.05–1.25 秒，说明局部爬山优化会带来更多的计算评估开销。
- 结论：若只需要满足阈值，未优化版本已经很高效；若要进一步提高满足子句数，`lab3_opt` 的局部爬山优化确实有明显收益，但时间成本更大。

任务二结论：
- `lab3` 版本的 METRIC-TSP 采用 MST+DFS 的 2-近似算法，并用全排列暴力求解最优回路，适合小规模城市测试。
- `lab3_opt` 版本在保持同样近似算法的基础上，对最优回路求解部分增加了回溯剪枝。
- 这样优化的好处是：在搜索最优回路时，若当前路径代价已经不可能优于已知最优解，就可以提前剪掉后续排列，减少搜索空间。
- 因此，`lab3_opt` 的 TSP 优化侧重于加速精确最优回路搜索，而不是改变近似回路本身。
- 实际结果说明：两者都能正确输出 MST、DFS 近似回路和最优回路；`lab3_opt` 的回溯剪枝使精确求解对小规模图的搜索更高效，尤其在城市数增加时，这种剪枝效果会更加明显。
