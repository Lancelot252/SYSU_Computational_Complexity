\section{摘要}

本实验旨在实现NP完全性理论中的多项式时间归约算法，具体包括：

    \item 任务2：实现3SAT问题到独立集问题（Independent Set）的多项式时间归约

通过本实验，深入理解NP完全性理论、多项式时间归约的概念和方法，掌握如何证明问题的NP完全性。

\section{问题描述}

\subsection{3SAT问题定义}

% TODO: 填写3SAT问题的形式化定义
3SAT问题是布尔可满足性问题的一个重要子类。给定一个合取范式（CNF）形式的布尔公式$\phi$，其中每个子句恰好包含3个文字，判断是否存在一个真值赋值使得$\phi$为真。

\textbf{形式化定义}：

设$\phi = C_1 \land C_2 \land \cdots \land C_m$，其中每个子句$C_i = (l_{i1} \lor l_{i2} \lor l_{i3})$，$l_{ij}$为文字（变量$x_k$或其否定$\neg x_k$）。

\textbf{问题}：是否存在赋值$\sigma: \{x_1, \ldots, x_n\} \to \{0, 1\}$使得$\phi(\sigma) = 1$？

\subsection{节点覆盖问题定义}

% TODO: 填写节点覆盖问题的形式化定义
节点覆盖问题（Vertex Cover Problem）是一个经典的图论问题。

\textbf{形式化定义}：

给定无向图$G = (V, E)$和正整数$k$，判断是否存在$V$的子集$V' \subseteq V$，满足：
\begin{enumerate}
    \item $|V'| \leq k$
    \item 对于每条边$(u, v) \in E$，至少有一个端点在$V'$中（即$u \in V'$或$v \in V'$）
\end{enumerate}

\subsection{独立集问题定义}

% TODO: 填写独立集问题的形式化定义
独立集问题（Independent Set Problem）同样是图论中的经典问题。

\textbf{形式化定义}：

给定无向图$G = (V, E)$和正整数$k$，判断是否存在$V$的子集$V' \subseteq V$，满足：
\begin{enumerate}
    \item $|V'| \geq k$
    \item 对于任意两个顶点$u, v \in V'$，有$(u, v) \notin E$（即$V'$中的顶点两两不相邻）
\end{enumerate}

\subsection{归约关系}

% TODO: 说明归约的理论基础
本实验需要实现以下多项式时间归约：
\begin{itemize}
    \item $3SAT \leq_p VertexCover$
    \item $3SAT \leq_p IndependentSet$
\end{itemize}

这些归约证明了节点覆盖问题和独立集问题都是NP完全的。

\subsection{任务1：3SAT $\to$ 节点覆盖归约算法}

本算法将3SAT实例$\phi$在多项式时间内归约为一个图$G$和整数$k$，使得$\phi$可满足当且仅当$G$存在一个大小不超过$k$的节点覆盖。

\subsubsection{归约思路}

给定一个3SAT公式$\phi = C_1 \land C_2 \land \cdots \land C_m$，其中每个子句$C_i = (l_{i1} \lor l_{i2} \lor l_{i3})$包含恰好三个文字。

\textbf{核心思想}：将每个子句的三种可能满足方式映射为图中的三角形结构，并通过冲突边确保变量赋值的一致性。每个三角形必须覆盖其中两条边才能用最少的节点，这恰好要求我们从三个文字中选择两个``假''文字，留下一个``真''文字来满足子句。冲突边则保证一个变量及其否定不能同时为真。

\subsubsection{构造方法}

设3SAT实例$\phi$包含$m$个子句和$n$个变量。

\textbf{步骤1：子句构件（三角形）}

对于每个子句$C_i = (l_{i1} \lor l_{i2} \lor l_{i3})$，在图中创建三个顶点，分别标记为$C_i{:}l_{i1}$、$C_i{:}l_{i2}$、$C_i{:}l_{i3}$，并在它们之间两两连边形成三角形。该三角形保证任意顶点覆盖必须至少选取其中两个顶点。

\textbf{步骤2：冲突边}

对于任意两个分别位于不同子句的顶点，如果它们对应的文字互为否定（即一个为$x$，另一个为$\neg x$），则在它们之间添加一条冲突边。冲突边确保了变量赋值的一致性：一个变量及其否定不能同时为真。

\textbf{步骤3：参数设置}

设定覆盖大小$k = 2m$（$m$为子句数量）。因为每个三角形至少需要2个顶点来覆盖其三条边，所以$k = 2m$是最小可能的节点覆盖大小。

\textbf{构造规模}：图$G$包含$|V| = 3m$个顶点，$|E| = 3m + e_c$条边（其中$e_c$为冲突边数量，至多为$O(m^2)$）。整个构造可在多项式时间内完成。

\subsubsection{正确性证明}

\textbf{定理}：3SAT公式$\phi$可满足 $\Leftrightarrow$ 归约得到的图$G$存在大小不超过$k = 2m$的节点覆盖。

\textbf{证明（$\Rightarrow$方向）}：

假设$\phi$存在一个满足赋值$\tau$。按如下方式构造节点覆盖$S$：
\begin{enumerate}
    \item 对于每个子句$C_i$，赋值$\tau$使得子句中至少一个文字为真。在该子句对应的三角形中，选取\textbf{不为真}的两条文字对应的顶点加入$S$。
    \item 每个三角形恰好选取2个顶点，因此$|S| = 2m = k$。
\end{enumerate}

验证$S$是有效的节点覆盖：
\begin{itemize}
    \item 对于三角形边：每个三角形的三条边中，选取的2个顶点覆盖了所有三条边（每条边至少有一个被选端点）。
    \item 对于冲突边：设冲突边连接顶点$v(x)$和$v(\neg x)$。在赋值$\tau$下，$x$和$\neg x$不能同时为真，因此至少有一个文字为假，其对应顶点被选入$S$。故冲突边被覆盖。
\end{itemize}

\textbf{证明（$\Leftarrow$方向）}：

假设图$G$存在一个大小不超过$k = 2m$的节点覆盖$S$。

\begin{enumerate}
    \item 每个三角形至少需要2个顶点来覆盖其三条边。图中共有$m$个互不相交的三角形（冲突边跨三角形连接，但三角形内部边仅由该三角形顶点覆盖）。因此$|S| \geq 2m$，结合$|S| \leq k = 2m$，得$|S| = 2m$，且每个三角形\textbf{恰好}有2个顶点在$S$中。
    \item 对于每个三角形，未被$S$选取的唯一顶点对应的文字被设定为``真''。据此定义变量赋值$\tau$：
    \begin{itemize}
        \item 若存在未被选取的顶点标记为文字$x$，则设$\tau(x) = \text{True}$。
        \item 若存在未被选取的顶点标记为文字$\neg x$，则设$\tau(x) = \text{False}$。
        \item 若变量$x$相关的所有顶点均被选取，则任意设定$\tau(x)$。
    \end{itemize}
    \item \textbf{赋值一致性}：假设矛盾，即存在顶点$v(x)$和$v(\neg x)$都未被$S$选取。由于它们之间的冲突边未被覆盖（两个端点都不在$S$中），与$S$为节点覆盖矛盾。因此赋值$\tau$的定义是良定义的。
    \item \textbf{公式可满足}：每个子句三角形中恰有一个顶点不在$S$中，该顶点对应的文字在该子句中为真，因此每个子句均被满足。故$\phi$在$\tau$下为真。
\end{enumerate}

综上，归约正确性得证。该归约的时间复杂度为$O(m^2)$（主要来自冲突边的添加），属于多项式时间归约。

\subsection{任务2：3SAT $\to$ 独立集归约算法}

本算法将3SAT实例$\phi$在多项式时间内归约为一个图$G$和整数$n$，使得$\phi$可满足当且仅当$G$存在一个大小至少为$n$的独立集。

\subsubsection{归约思路}

给定一个3SAT公式$\phi = C_1 \land C_2 \land \cdots \land C_m$，其中每个子句$C_i = (l_{i1} \lor l_{i2} \lor l_{i3})$包含恰好三个文字。

\textbf{核心思想}：将每个子句映射为图中的三角形结构，三角形的三个顶点分别代表子句中的三个文字。从每个三角形中选择恰好一个顶点构成独立集，该顶点对应的文字即为该子句中被赋值为真的文字。由于独立集中任意两点不能有边相连，因此需要添加冲突边来确保变量赋值的一致性——对于同一变量的正文字和负文字对应的顶点之间添加边，防止它们同时被选入独立集。

\subsubsection{构造方法}

设3SAT实例$\phi$包含$m$个子句和$n$个变量。

\textbf{步骤1：子句构件（三角形）}

对于每个子句$C_i = (l_{i1} \lor l_{i2} \lor l_{i3})$，在图中创建三个顶点，分别标记为文字$l_{i1}$、$l_{i2}$、$l_{i3}$，并在它们之间两两连边形成三角形。该三角形保证任意独立集最多只能从其中选取一个顶点，对应每个子句恰好选择一个文字为真。

\textbf{步骤2：冲突边}

对于任意两个分别位于不同子句的顶点，如果它们对应的文字互为否定（即一个为$x$，另一个为$\neg x$），则在它们之间添加一条冲突边。冲突边确保独立集中不能同时包含$x$和$\neg x$，从而保证变量赋值的一致性。

\textbf{步骤3：参数设置}

设定独立集大小$n = m$（$m$为子句数量）。因为每个三角形最多贡献1个顶点到独立集，所以$n = m$是最大可能的独立集大小。

\textbf{构造规模}：图$G$包含$|V| = 3m$个顶点，$|E| = 3m + e_c$条边（其中$e_c$为冲突边数量，至多为$O(m^2)$）。整个构造可在多项式时间内完成。

\subsubsection{正确性证明}

\textbf{定理}：3SAT公式$\phi$可满足 $\Leftrightarrow$ 归约得到的图$G$存在大小至少为$n = m$的独立集。

\textbf{证明（$\Rightarrow$方向）}：

假设$\phi$存在一个满足赋值$\tau$。按如下方式构造独立集$S$：
\begin{enumerate}
    \item 对于每个子句$C_i$，赋值$\tau$使得子句中至少一个文字为真。从该子句对应的三角形中，选取任意一个为真的文字对应的顶点加入$S$。
    \item 每个三角形恰好选取1个顶点，因此$|S| = m = n$。
\end{enumerate}

验证$S$是有效的独立集：
\begin{itemize}
    \item 对于三角形内部边：每个三角形中只选取了1个顶点，因此三角形内部任意边最多只有一个端点在$S$中，不会违反独立集条件。
    \item 对于冲突边：设冲突边连接顶点$v(x)$和$v(\neg x)$。在赋值$\tau$下，$x$和$\neg x$不能同时为真，因此最多只有一个文字为真的顶点被选入$S$。故冲突边的两个端点不会同时出现在$S$中。
\end{itemize}
因此$S$是一个大小为$m$的独立集，满足$|S| \geq n$。

\textbf{证明（$\Leftarrow$方向）}：

假设图$G$存在一个大小至少为$n = m$的独立集$S$。

\begin{enumerate}
    \item 每个三角形最多只能贡献1个顶点到独立集（因为三角形内部有三条边，任意两个顶点之间都有边）。图中共有$m$个互不相交的三角形，因此$|S| \leq m$。结合$|S| \geq n = m$，得$|S| = m$，且每个三角形\textbf{恰好}有1个顶点在$S$中。
    \item 对于每个三角形，被$S$选取的唯一顶点对应的文字被设定为``真''。据此定义变量赋值$\tau$：
    \begin{itemize}
        \item 若被选取的顶点标记为文字$x$，则设$\tau(x) = \text{True}$。
        \item 若被选取的顶点标记为文字$\neg x$，则设$\tau(x) = \text{False}$。
    \end{itemize}
    \item \textbf{赋值一致性}：假设矛盾，即存在变量$x$同时被赋值True和False。这意味着存在顶点$v(x)$和$v(\neg x)$同时被选入$S$。但由于它们之间存在冲突边，这与$S$为独立集矛盾。因此赋值$\tau$的定义是良定义的。
    \item \textbf{公式可满足}：每个子句三角形中恰有一个顶点在$S$中，该顶点对应的文字在该子句中为真，因此每个子句均被满足。故$\phi$在$\tau$下为真。
\end{enumerate}

综上，归约正确性得证。该归约的时间复杂度为$O(m^2)$（主要来自冲突边的添加），属于多项式时间归约。

\subsection{任务1关键代码说明}

\subsubsection{归约函数}

归约函数 \texttt{reduce\_3sat\_to\_vertex\_cover} 实现了3SAT到节点覆盖的多项式时间归约。核心流程为：遍历每个子句创建三角形顶点，然后为所有互补文字对添加冲突边。

\begin{lstlisting}[language=Python]
# 位于 src/reductions/vertex_cover.py
def reduce_3sat_to_vertex_cover(formula: Formula3SAT) -> Tuple[Graph, int]:
    graph = Graph()
    m = formula.get_clause_count()
    k = 2 * m  # 从每个三角形的2个顶点构成覆盖

    # 记录每种文字对应的节点ID列表
    literal_nodes = {}

    for i, clause in enumerate(formula):
        for j, literal in enumerate(clause):
            label = format_node_label(i + 1, str(literal))
            graph.add_node(label)
            key = (literal.variable, literal.negated)
            if key not in literal_nodes:
                literal_nodes[key] = []
            literal_nodes[key].append(i * 3 + j)

        # 创建子句三角形边
        base = i * 3
        graph.add_edge(base, base + 1)
        graph.add_edge(base + 1, base + 2)
        graph.add_edge(base, base + 2)

    # 为同变量的正负文字节点添加冲突边
    for var in formula.get_variables():
        pos_nodes = literal_nodes.get((var, False), [])
        neg_nodes = literal_nodes.get((var, True), [])
        for pos_node in pos_nodes:
            for neg_node in neg_nodes:
                if not graph.has_edge(pos_node, neg_node):
                    graph.add_edge(pos_node, neg_node)

    return graph, k
\end{lstlisting}

\subsubsection{解的转换}

从节点覆盖解提取满足赋值的函数 \texttt{extract\_satisfying\_assignment} 基于以下原理：在大小为$k = 2m$的节点覆盖中，每个子句三角形恰好有2个顶点被选取，未被选取的那个顶点对应一个在该子句中为真的文字。

# 位于 src/reductions/vertex_cover.py
def extract_satisfying_assignment(formula, graph, vertex_cover):
    assignment = {}
    m = formula.get_clause_count()

    for i, clause in enumerate(formula):
        for j, literal in enumerate(clause):
            node_id = i * 3 + j
            if node_id not in vertex_cover:
                # 此顶点未被覆盖 → 对应文字取值为真
                if literal.negated:
                    assignment[literal.variable] = False
                else:
                    assignment[literal.variable] = True

    # 对未赋值的变量默认为True（取值不影响公式可满足性）
    for var in formula.get_variables():
        if var not in assignment:
            assignment[var] = True

    return assignment

\subsubsection{归约函数}
\textbf{原理：}
\begin{enumerate}

def reduce_3sat_to_independent_set(formula: Formula3SAT) -> Tuple[Graph, int]:
    graph = Graph()
    global _node_clause_info
    _node_clause_info = {}
    
    # 记录所有正/负文字对应的节点列表
    pos_nodes: Dict[str, List[int]] = {}   
    neg_nodes: Dict[str, List[int]] = {}   
    
    # 为每个子句创建三角形
    for clause_idx, clause in enumerate(formula):
        triangle_nodes = []
        for lit in clause:
            label = str(lit)  # "x" 或 "¬x"
            node_id = graph.add_node(label)
            triangle_nodes.append(node_id)
            # 记录节点所属子句索引和文字信息
            _node_clause_info[node_id] = (clause_idx, label)
            
            # 根据正/负记录节点
            if lit.negated:
                neg_nodes.setdefault(lit.variable, []).append(node_id)
            else:
                pos_nodes.setdefault(lit.variable, []).append(node_id)
        
        # 创建三角形的三条边
        graph.add_edge(triangle_nodes[0], triangle_nodes[1])
        graph.add_edge(triangle_nodes[1], triangle_nodes[2])
        graph.add_edge(triangle_nodes[0], triangle_nodes[2])
    
    # 对于每对互为否定的文字，在对应的节点之间添加边
    for var in formula.variables:
        pos_list = pos_nodes.get(var, [])
        neg_list = neg_nodes.get(var, [])
        for p_node in pos_list:
            for n_node in neg_list:
                if not graph.has_edge(p_node, n_node):
                    graph.add_edge(p_node, n_node)
    
    # 计算独立集大小 = 子句数
    n = len(formula)
    
    return graph, n
\end{lstlisting}

\subsubsection{解的转换}
\textbf{原理：}
从独立集解中提取满足赋值。当归约后的图存在大小为n的独立集时，此函数用于从独立集中提取原始3SAT公式的满足赋值。
\begin{enumerate}
    \item 独立集中的每个节点代表一个文字
    \item 如果该文字为正（如x），则令该变量为True
    \item 如果该文字为负（如¬x），则令该变量为False
    \item 对于独立集中未出现的变量，可以任意赋值（比如True）
\end{enumerate}

\begin{lstlisting}[language=Python]
def extract_satisfying_assignment(formula: Formula3SAT, graph: Graph, 
                                  independent_set: set) -> dict:
    """
    从独立集解中提取满足赋值
    
    当归约后的图存在大小为n的独立集时，此函数用于
    从独立集中提取原始3SAT公式的满足赋值。
    
    """
    assignment: Dict[str, bool] = {}
    
    # 从独立集中提取赋值
    for node_id in independent_set:
        label = graph.get_label(node_id)
        if not label:
            continue
        
        # 判断是正文字还是负文字
        if label.startswith('¬'):
            var = label[1:]
            # 负文字在独立集中,变量为 False
            if var not in assignment:
                assignment[var] = False
        else:
            var = label
            # 正文字在独立集中,变量为 True
            if var not in assignment:
                assignment[var] = True
    
    # 给独立集中未出现的变量赋予默认值（True）
    for var in formula.variables:
        if var not in assignment:
            assignment[var] = True
    
    return assignment
\end{lstlisting}

        \item 解决方案：查阅Sipser教材和课程课件，确认两种构造方式均为有效的多项式时间归约。简化方案（$k = 2m$）避免了变量构件的引入，减少了图的顶点数（从$2n + 3m$降至$3m$），同时保持了归约的正确性——三角形结构保证每个子句恰好选出2个顶点，冲突边保证变量赋值的一致性。
    \end{itemize}
    \item 问题2：从节点覆盖中提取满足赋值时，需要正确处理变量未被任何三角形顶点``选中为真''的情况——当某个变量的所有正负文字对应顶点都在节点覆盖中时，该变量未被任何已确定赋值的文字引用。
    \begin{itemize}
        \item 解决方案：对于这种情况，可以任意赋值（如默认为True），因为这不会影响公式的可满足性——该变量的任何赋值都已经与其他子句的满足状态无关。在实现中，对未被赋值的变量统一默认为True。
    \end{itemize}
\end{itemize}

% TODO: 人员C填写任务2遇到的问题
\textbf{任务2（人员C）}：
\begin{itemize}
    \item 问题1：理解独立集归约中$n$值的含义——在独立集问题中，目标是是否存在大小\textbf{至少}为$n$的独立集，这与节点覆盖的大小\textbf{不超过}$k$的方向相反。初看实现时容易混淆参数含义。
    \begin{itemize}
        \item 解决方案：查阅课件确认独立集问题的定义（$|S| \geq n$），明确归约中的$n$等于子句数$m$，表示每个三角形恰好贡献一个顶点到独立集中。
    \end{itemize}
    \item 问题2：曲线冲突边的绘制优化——在可视化过程中，三角形内部边和冲突边重叠导致图片混乱，特别是多条冲突边交叉时难以辨认。
    \begin{itemize}
        \item 解决方案：采用不同的线型和颜色区分两类边——三角形内部边使用黑色实线，冲突边使用不同颜色的曲线（通过\texttt{FancyArrowPatch}的\texttt{arc3}连接样式实现），并根据冲突边的Y坐标位置决定曲线的弯曲方向（上方边向上拱、下方边向下拱），有效避免了边的重叠遮挡。
    \end{itemize}
\end{itemize}
